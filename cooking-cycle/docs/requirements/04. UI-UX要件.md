# UI/UX要件（概要）

## 基本方針

- **モバイルファースト：** スーパーやキッチンでの利用を想定
- **PC対応：** 初期データ入力など、まとめて登録する操作
- **PWA対応：** スマホでホーム画面にアイコンを配置して起動可能
- **オフライン機能：** 不要（オンライン前提）

## 画面一覧・画面構成（検討事項）

### 検討が必要な画面

以下は検討が必要な画面のリストです。**使い勝手を考慮して、PCとスマホで求められる機能の違いを踏まえて設計します。** 現時点では決めつけず、実際の利用シーンを踏まえて検討していきます。

#### 基本画面

1. **ホーム画面**
	- **検討事項：** スマホとPCで必要な情報が違う可能性
	- **スマホ：** スーパーやキッチンでの利用を想定、すぐにアクセスできる機能
	- **PC：** 初期データ入力やまとめて登録する操作

2. **冷凍庫ストック一覧**
	- **検討事項：** 消費推奨アラートの表示方法、ソート・フィルター機能
	- **スマホ：** 片手で操作できる、一目で分かる表示
	- **PC：** 一覧性を重視、まとめて操作できる機能

3. **ストック登録・編集画面**
	- **検討事項：** 入力項目、入力方法、バリデーション
	- **スマホ：** 高速入力UI、最小限の操作
	- **PC：** フォーム入力がしやすい、一括登録機能

4. **レシピ一覧**
	- **検討事項：** 表示方法（カード、リスト）、検索・フィルターのUI
	- **スマホ：** スワイプ操作、タップで詳細表示
	- **PC：** 一覧性を重視、複数選択・一括操作

5. **レシピ詳細画面**
	- **検討事項：** 表示する情報、YouTube動画の表示方法
	- **スマホ：** キッチンで見やすい、大きなボタン
	- **PC：** 詳細情報を表示、編集しやすい

6. **レシピ登録・編集画面**
	- **検討事項：** 入力項目、入力方法、アレンジレシピの登録方法
	- **スマホ：** 最小限の入力、簡単な操作
	- **PC：** 詳細な入力、一括登録、複雑な操作

7. **レシピ検索・フィルター画面**
	- **検討事項：** 検索UI、フィルターUI、結果表示
	- **スマホ：** スーパーで使いやすい、簡単な操作
	- **PC：** 複雑な検索条件、高度なフィルター

### PCとスマホで求められる機能の違い

#### スマホ（スーパー・キッチンでの利用）

- **片手操作：** 片手で操作できるUI
- **高速確認：** すぐに情報を確認できる
- **最小限の入力：** 簡単な操作で登録・更新
- **大きなボタン：** 手が汚れていても操作できる
- **見やすい表示：** 一目で分かる情報表示

#### PC（初期データ入力・まとめて登録）

- **フォーム入力：** キーボードで入力しやすい
- **一括操作：** 複数のレシピをまとめて登録
- **詳細な入力：** 複雑な情報も入力しやすい
- **一覧性：** 多くの情報を一度に表示
- **編集しやすさ：** マウス・キーボードでの操作

## モバイルファーストの操作フロー

### スーパーでの利用

- **操作：** 片手で操作できる
- **表示：** 冷凍庫ストック一覧が見やすい
- **入力：** 最小限の操作で確認できる

### キッチンでの利用

- **操作：** 手が汚れていても操作できる（大きなボタン）
- **表示：** レシピ詳細が見やすい
- **再生：** YouTube動画へのリンクが分かりやすい

### PCでの利用

- **操作：** マウス・キーボードでの操作
- **入力：** フォーム入力がしやすい
- **一括登録：** 複数のレシピをまとめて登録できる

## PWA要件

- **マニフェスト：** `public/manifest.json` を配置
- **アイコン：** `public/icons/` に配置（192x192、512x512推奨）
- **ホーム画面追加：** スマホでホーム画面にアイコンを設置できる
- **オフライン機能：** 不要（オンライン前提）

## レスポンシブ対応

- **スマホ：** 320px〜
- **PC：** 1024px〜

## ダークモード

- ライト/ダークモードの切り替え機能を実装
- システム設定に追従するオプションも推奨
