# 開発フェーズ・実装計画

## 基本方針

- **長期的な視点：** 長いこと使いながら育てていくアプリ
- **変更に強い設計：** テーブル定義も仕様も今後変わる可能性がある
- **段階的な開発：** 初期段階ではとりあえず機能を実装、今後育てていく
- **アイデアの記録：** 今後開発したいアイデアもメモしておく

## Phase 1の範囲定義

### Phase 1: 基礎機能

**目標：** 基本的なレシピ管理とストック管理ができる

1. **レシピ管理**
	- 基本レシピの登録・編集・削除
	- レシピ一覧表示
	- レシピ詳細表示
	- YouTube URLの保存・リンク

2. **ストック管理**
	- ストックの登録・編集・削除
	- ストック一覧表示
	- 冷凍開始日の記録
	- 消費推奨アラート（1ヶ月以上経過）

3. **基本的な検索**
	- レシピ一覧の表示
	- ストック優先フィルター（ワンクリック）

4. **認証**
	- Google認証

5. **PWA対応**
	- ホーム画面にアイコンを追加できる

### Phase 2: 拡張機能

**目標：** より便利に使える機能を追加

1. **アレンジレシピ**
	- 基本レシピの子としてアレンジレシピを登録
	- アレンジレシピ一覧表示
	- **重要：** アレンジレシピはデータモデル設計で事前に念頭に置いておく必要がある
		- 基本レシピとアレンジレシピの継承構造
		- アレンジレシピの追加・変更材料の管理方法
		- 手順の継承方法
		- データモデル設計時に十分に検討する

2. **タグ機能**
	- レシピにタグを付けられる
	- タグでの検索・フィルター

3. **高度な検索**
	- 複数タグのAND/OR/NOT検索
	- 作り置きからの提案

4. **場所メモ**
	- ストックに場所メモを追加

5. **タイムスタンプ**
	- YouTube動画のタイムスタンプを保存・ジャンプ

### Phase 3: 将来的な拡張（アイデア）

**今後開発したい機能（メモ）**

- YouTube Data API連携
- レシピの画像アップロード
- レシピの評価・お気に入り
- 献立カレンダー
- 買い物リスト機能
- レシピの共有機能

## 開発の進め方

1. **初期段階：** とりあえず機能を実装
2. **使ってみる：** 実際に使ってみて問題点を発見
3. **改善：** 問題点を修正・機能を追加
4. **繰り返し：** 長い目で見て育てていく

## 注意事項

- テーブル定義も仕様も今後変わる可能性がある
- 変更に強い設計を心がける
- テストをしっかり書いて、変更に強いコードにする
- 使っていく中で育てていく

## データモデル設計時の重要な検討事項

### アレンジレシピの設計

**アレンジレシピは後から追加する機能だが、データモデル設計時に事前に念頭に置いておく必要がある。**

**検討すべき点：**
- 基本レシピとアレンジレシピの継承構造をどう設計するか
- アレンジレシピの追加・変更材料をどう管理するか
- 手順の継承方法（基本レシピの手順を継承するのか、完全に別にするのか）
- アレンジレシピからさらにアレンジレシピを作れるか（多階層の可能性）
- 検索・フィルター時に基本レシピとアレンジレシピをどう扱うか

**設計方針：**
- Phase 1では実装しないが、データモデル設計時に拡張性を考慮する
- 後から追加する際に大きな変更が不要な設計にする
