# 設計フェーズアウトプット一覧

設計フェーズで作成するドキュメントの一覧です。

## 01. 機能一覧
- 各機能の詳細な機能仕様
- 機能の優先順位（Phase 1 / Phase 1.5 / Phase 2 / Phase 3）
- 機能間の依存関係
- 機能の入力・出力・処理内容

## 02. 画面設計書
- 各画面のレイアウト設計
- UIコンポーネントの配置
- 画面遷移図（詳細版）
- モバイルファーストのレスポンシブ設計
- インタラクション設計（ボタン、フォーム、モーダル等）

## 03. API設計書
- REST APIエンドポイント一覧
- リクエスト・レスポンス仕様
- エラーハンドリング仕様
- 認証・認可の仕様
- Next.js App Routerのルーティング設計

## 04. テーブル定義書
- データベーススキーマ設計
- テーブル一覧と各テーブルのカラム定義
- 主キー・外部キー・インデックス定義
- 制約条件（NOT NULL、UNIQUE等）
- リレーションシップの詳細
- マイグレーション戦略

## 05. DFD（データフローダイアグラム）
- システム全体のデータフロー
- 各プロセスのデータ入出力
- データストアとの関係
- 外部エンティティとの関係

## 06. インフラ仕様書
- ローカル開発環境の構成（Docker Compose）

	- RDSデータベース仕様
	- S3バケット構成
	- ネットワーク構成（VPC、サブネット、セキュリティグループ）
	- ロードバランサー設定
- デプロイメント手順
- バックアップ・災害復旧戦略

## 07. 単体テスト仕様書
- テスト対象機能一覧
- 各機能のテストケース
- テストデータ設計
- カバレッジ目標

## 08. 統合テスト仕様書（E2Eテスト）
- E2Eテストシナリオ一覧
- 主要ユースケースのテストフロー

## 09. 設計レビュー記録
- 外部レビューによる評価内容
