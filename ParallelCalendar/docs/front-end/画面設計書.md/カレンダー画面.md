# カレンダー画面

## 画面概要

並列カレンダー表示画面。

認証突破済みユーザーのみアクセス可能。

各ユーザーのカレンダーが並列に表示される。

今後、右サイドバーを追加予定

v1.0では、ページ本体と、右サイドバーのみを実装する。

ページ本体では、全アロケーションの3日分のカレンダーを表示する。

```tsx
<ParallelCalendar />
```

右サイドバーでは、ページ本体で選択しているタスクと時間帯の詳細を表示し、編集可能とする。

```tsx
<ParallelCalendarSidebar />
```

### URL

/parallel-calendar のみアクセス可能。


## 画面要素

1. 共通ヘッダー
2. ページ本体（CalendarPageコンポーネント）
   1. 全アロケーションの3日分のカレンダーを表示する。
   2. ParallelCalendarコンポーネントのパラメータ
      1. TimeSlotをparallel-calendarのパラメータに設定可能とする。
      2. resourceなどparallel-calendarの内部仕様に影響するものは、parallel-calendarの内部仕様を優先する。
         1. アロケーションに対応すresource、開始日など
      3. 複数のアロケーションでも、同じタスクの時間帯（time-slot）は同じタスクとしてまとめて扱う。
   3. ページ本体
      1. データ
         1. 日付範囲
            1. 開始日
            2. 終了日
         2. タスク
            1. 複数の時間帯（time-slot）
               1. アロケーションごと。
               2. v0.1では、PlanとDoの2つのアロケーションを1つのタスクとする。
               3. Planのみ、Doのみ、PlanとDoの両方の時間帯（time-slot）を持つタスクを表示する。
3. 右サイドバー
   1. ページ本体で選択しているタスクと時間帯（time-slot）の詳細を表示し、編集可能とする。
   2. 予定の詳細な内容を追加・変更・削除可能なダイアログとして定義する。
4. 

```tsx
{/* <CalendarLeftSidebar /> */}
<CalendarPage />
<CalendarRightSidebar />
```


## イベント

1. ページ本体
   1. 時間帯（time-slot）を選択すると、タスク単位で右サイドバーに詳細が表示される。
   2. 時間帯（time-slot）を編集すると、タスク単位で右サイドバーに詳細が反映される。
2. 右サイドバー
   1. タスクを編集すると、タスク単位でページ本体のカレンダーが更新される。
   2. 編集差分がなければ、他のタスクを選択すると、右サイドバーの詳細が切り替わる。
   3. 編集差分がある場合、他のタスクを選択すると、編集内容を保存するか、キャンセルするかを選択する警告が表示される。
3. ヘッダー
   1. ログアウトボタンをクリックすると、ログイン画面に遷移する。
   2. 日付選択ボタンをクリックすると、日付選択モーダルが表示される。
   3. 日付選択モーダルで日付を選択すると、ページ本体のカレンダーが更新される。
   4. 日付選択モーダルで閉じるボタンをクリックすると、日付選択モーダルが閉じられる。
