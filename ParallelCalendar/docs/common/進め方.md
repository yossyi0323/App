大まかな進め方
1. react big calendarで想定しているUIが実現できるかをmockでテキトーなデータで確認する
2. 最低限のDDLでテーブル用意して、各リソースのバックエンドのCRUDのREST APIを作る
    1. HTTP通信の基礎とかは勉強したい
    2. 1回はちゃとボイラープレートなテンプレのコードを書く手間を味わって、自分でめんどくさいって思ってから、省力化してクリーンなコードにしたい。
    3. DB接続の部分も、ある程度共通化できるだろうが、まずはちゃんとめんどくさいことをやってから省力化する。
3. 最低限のUIを作る
    1. ボタン操作中心で、キーボードショートカットはそこまで充実させなくていいから、最低限動くものを作る。特に、各リソースのCRUDの疎通を優先する。
    2. taskとtimeSlotsのCRUD、バーティカルカレンダーのD&D、詳細画面での入力ができればOK。D&Dは多分ライブラリのほうの機能で入ってると思う（要検証）。
4. DockerコンテナをAWSにデプロイする
    1. 最低限アプリとしてCRUDが疎通できたタイミングで、一回デプロイする。v0.1みたいなもの。
5. 最低限のテストコードや、Google認証や、キーボードショートカットでの操作性向上とかのエンハンスを加える
    1. この段階まで来たらいったんRePlanしよう。
