# E2Eテストケース一覧

## テスト観点

### 1. 投稿作成機能
- [ ] TC-E2E-001: テキストを入力してSubmitボタンをクリックすると投稿が作成される
- [ ] TC-E2E-002: 空のテキストでSubmitすると投稿が作成されない
- [ ] TC-E2E-003: 投稿作成後、テキストエリアがクリアされる
- [ ] TC-E2E-004: 投稿作成中は「送信中...」と表示される
- [ ] TC-E2E-005: 作成した投稿が一覧の最上部に表示される

### 2. 投稿一覧表示機能
- [ ] TC-E2E-006: ページロード時に既存の投稿一覧が表示される
- [ ] TC-E2E-007: 投稿が新しい順に表示される
- [ ] TC-E2E-008: 各投稿にEditボタンとDeleteボタンが表示される
- [ ] TC-E2E-009: 投稿が0件の場合「投稿がありません」と表示される
- [ ] TC-E2E-010: 投稿には作成日時が表示される

### 3. 投稿編集機能
- [ ] TC-E2E-011: Editボタンをクリックするとテキストエリアに内容が読み込まれる
- [ ] TC-E2E-012: 編集モード中、ボタンが「Save」に変更される
- [ ] TC-E2E-013: 編集モード中、Cancelボタンが表示される
- [ ] TC-E2E-014: Saveボタンをクリックすると投稿が更新される
- [ ] TC-E2E-015: Cancelボタンをクリックすると編集がキャンセルされる
- [ ] TC-E2E-016: 更新後、リストに変更内容が反映される

### 4. 投稿削除機能
- [ ] TC-E2E-017: Deleteボタンをクリックすると確認ダイアログが表示される
- [ ] TC-E2E-018: 確認ダイアログでOKをクリックすると投稿が削除される
- [ ] TC-E2E-019: 確認ダイアログでキャンセルをクリックすると削除されない
- [ ] TC-E2E-020: 削除後、リストから該当の投稿が消える

### 5. 統合フロー
- [ ] TC-E2E-021: 投稿作成→編集→削除の一連のフローが正常に動作する
- [ ] TC-E2E-022: 複数の投稿を作成→それぞれ編集→削除が正常に動作する

## 実行方法

### 前提条件
- サーバーが起動していること（`http://localhost:8080`）
- Node.js 18以上がインストールされていること

### セットアップ
```bash
cd C:\Users\yoshi\Documents\App\minimum-sns-post-app2\e2e
npm install
npx playwright install chromium
```

### テスト実行
```bash
# 全テスト実行
npm test

# UIモードで実行（デバッグに便利）
npm run test:ui

# ヘッドフルモードで実行（ブラウザが見える）
npm run test:headed

# デバッグモード
npm run test:debug

# レポート表示
npm run report
```

## テスト結果の見方

- `playwright-report/index.html`: HTML形式のテストレポート
- スクリーンショット: テスト失敗時に自動保存
- トレース: 再実行時に自動保存（`trace.playwright.dev`で確認可能）
